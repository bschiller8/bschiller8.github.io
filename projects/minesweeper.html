Overview

What it is & what I focused on

Product

  • Playable Minesweeper in the browser
  • Left click reveals, right click flags (or your chosen control)
  • Clear win/loss feedback + reset

What I cared about

  • Reliable state management (tiles, mines, flags, revealed)
  • Edge cases (first click safety if you implemented it, bounds, repeat clicks)
  • Clean UI feedback (hover/reveal/flag states)

What I built

Quick, recruiter-friendly highlights.

Core mechanics

Grid generation, mine placement, number hints, flagging, and win/loss logic.

Reveal propagation

Flood-fill reveal behavior for empty tiles so the board opens naturally.

Interaction polish

Smooth feedback for hover/reveal/flag states and clear reset behavior.

Edge cases

Defensive checks for repeated clicks, bounds, and preventing invalid actions.

⌘ Implementation

Tech notes

How it works (high level).

State

  • 2D array of cell objects (mine, revealed, flagged, adjacentCount)
  • Global counters (mines, flags, revealed cells)
  • Game mode (playing / won / lost)

Algorithm

  • Mine placement (random unique positions)
  • Compute adjacent counts for each cell
  • Flood-fill reveal for zero-adjacent cells
✎ Reflection

Reflection

What I learned + what I’d do next.

What I learned

  • Game UI is basically state management with strong feedback loops.
  • Edge cases matter more than the “happy path.”

Next improvements

  • Add difficulty selector + timer + best time tracking.
  • Keyboard support + accessibility pass (focus states, ARIA).
