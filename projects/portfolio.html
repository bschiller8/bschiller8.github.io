Overview

Why I built this portfolio

Problem

  • Most portfolio templates felt generic or rigid.
  • I wanted full control over layout and interaction.
  • My UX work needed space to breathe and tell a story.

Solution

  • A custom layout system for projects and case studies.
  • Consistent components across every page.
  • Designed for quick scanning by recruiters.

Structure

How the site is organized

  • Homepage cards link to dedicated project pages.
  • UX case studies use a deeper, research-focused layout.
  • Dev projects use a lighter but consistent structure.

Design decisions

What I focused on visually

Hierarchy

Clear headings, spacing, and card layouts make pages easy to skim without losing important detail.

Consistency

Shared components (cards, badges, buttons) create a cohesive experience across the site.

Subtle motion

Lightweight reveal animations add polish without distraction.

Accessibility

  • Semantic HTML structure and landmarks.
  • Alt text on images and thumbnails.
  • Skip links and keyboard-friendly navigation.
  • Respects prefers-reduced-motion.

Performance

  • Static hosting for fast load times.
  • Lazy-loaded images where possible.
  • Minimal JavaScript for interaction only.
